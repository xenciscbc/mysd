**A. Create `.claude/agents/mysd-researcher.md` (FAGENT-01, per D-03):**

Frontmatter:
```yaml
---
model: claude-sonnet-4-5
description: Research agent. Investigates a specific dimension (Codebase, Domain, Architecture, or Pitfalls) and produces structured findings.
allowed-tools:
  - Read
  - Glob
  - Grep
  - Bash
  - WebFetch
---
```

Body structure:
- **Input**: Context JSON with `change_name`, `dimension` (one of: "codebase", "domain", "architecture", "pitfalls"), `topic`, `spec_files` array, `auto_mode` boolean
- **Step 1: Identify Research Scope** — Based on `dimension`:
  - `codebase`: Search existing code for patterns, dependencies, integration points relevant to topic
  - `domain`: Research domain concepts, best practices, industry standards relevant to topic
  - `architecture`: Analyze architectural implications, component interactions, scalability concerns
  - `pitfalls`: Identify common mistakes, anti-patterns, edge cases, failure modes
- **Step 2: Gather Evidence** — Read relevant files (codebase), search web (domain/pitfalls), analyze structure (architecture)
- **Step 3: Produce Findings** — Output structured report:
  ```
  ## Research: {dimension} — {topic}
  ### Key Findings
  - {finding 1}
  - {finding 2}
  ### Implications for Implementation
  - {implication}
  ### Risks/Concerns
  - {risk}
  ### Recommendations
  - {recommendation}
  ```
- **auto_mode handling**: If `auto_mode` is true, skip any interactive clarification and use best judgment for ambiguous areas

CRITICAL: Do NOT include Task tool in allowed-tools. Do NOT describe spawning other agents anywhere in the prompt.

**B. Create `.claude/agents/mysd-advisor.md` (FAGENT-02):**

Frontmatter:
```yaml
---
model: claude-sonnet-4-5
description: Advisor agent. Analyzes gray areas and produces structured comparison tables with trade-off analysis.
allowed-tools:
  - Read
  - Glob
  - Grep
  - Bash
---
```

Body structure:
- **Input**: Context JSON with `change_name`, `gray_area` (description of ambiguous area), `research_findings` (from researcher), `auto_mode` boolean
- **Step 1: Analyze the Gray Area** — Read relevant spec files and codebase to understand the ambiguity
- **Step 2: Identify Options** — List 2-4 viable approaches
- **Step 3: Produce Comparison Table** — Output structured analysis:
  ```
  ## Analysis: {gray_area}

  | Criterion | Option A: {name} | Option B: {name} | Option C: {name} |
  |-----------|-------------------|-------------------|-------------------|
  | Complexity | {low/medium/high} | ... | ... |
  | Risk | {low/medium/high} | ... | ... |
  | Alignment with spec | {description} | ... | ... |
  | Maintenance burden | {description} | ... | ... |

  ### Recommendation
  **Option {X}** because {rationale}.

  ### Trade-offs to Accept
  - {trade-off 1}
  ```
- **auto_mode handling**: If true, output recommendation without asking for user preference

CRITICAL: No Task tool. No subagent spawning.

**C. Create `.claude/agents/mysd-proposal-writer.md` (FAGENT-03):**

Frontmatter:
```yaml
---
model: claude-sonnet-4-5
description: Proposal writer agent. Creates or updates proposal.md for a change based on discussion conclusions or user requirements.
allowed-tools:
  - Read
  - Write
  - Edit
  - Bash
---
```

Body structure:
- **Input**: Context JSON with `change_name`, `conclusions` (from discussion or user input), `existing_proposal` (body text if updating), `auto_mode` boolean
- **Step 1: Read Existing Context** — If `existing_proposal` provided, read `.specs/changes/{change_name}/proposal.md` to understand current state
- **Step 2: Draft Proposal** — Write/update proposal.md with sections: Summary, Motivation, Scope (In Scope / Out of Scope), Success Criteria
  - If creating new: scaffold from conclusions
  - If updating: merge conclusions into existing proposal, preserving unchanged sections
- **Step 3: Write File** — Write to `.specs/changes/{change_name}/proposal.md`
- **Step 4: Run State Transition** — `mysd propose {change_name}` (only if creating new, not updating)
- **auto_mode handling**: If true, write directly without asking user to review draft

CRITICAL: No Task tool. No subagent spawning.
