**A. Create `.claude/commands/mysd-apply.md` (D-22, FAGENT-07 per-task spawn):**

Frontmatter:
```yaml
---
model: claude-sonnet-4-5
description: Execute pending tasks with mandatory alignment gate. Supports single (sequential per-task) and wave (parallel per-task) modes. Usage: /mysd:apply [--auto]
allowed-tools:
  - Bash
  - Read
  - Write
  - Edit
  - Task
---
```

Body — `/mysd:apply`:

Step 1: Parse Arguments
- Check `$ARGUMENTS` for `--auto`. Set `auto_mode` accordingly.

Step 2: Get Execution Context
- Run `mysd execute --context-only` (binary subcommand unchanged per RESEARCH.md recommendation)
- Parse JSON: change_name, must_items, should_items, may_items, tasks, pending_tasks, tdd_mode, atomic_commits, execution_mode, agent_count, wave_groups, worktree_dir, auto_mode (from binary)
- If `--auto` was parsed in Step 1, override auto_mode to true

Step 3: Execute Based on Mode

**Single Mode (execution_mode == "single"):**
For each task in pending_tasks (sequential, one at a time):
  Use the Task tool to invoke `mysd-executor`:
  ```
  Task: Execute task T{task.id}: {task.name}
  Agent: mysd-executor
  Context: {
    "change_name": "{change_name}",
    "must_items": [...],
    "should_items": [...],
    "may_items": [...],
    "tasks": [...],
    "assigned_task": {current task object},
    "tdd_mode": {tdd_mode},
    "atomic_commits": {atomic_commits},
    "auto_mode": {auto_mode}
  }
  ```
  Wait for completion before spawning next executor.

**Wave Mode (execution_mode == "wave"):**
Process wave_groups sequentially. Within each wave, spawn executors in parallel:
  For each wave in wave_groups:
    For each task in wave (parallel):
      Use the Task tool to invoke `mysd-executor`:
      ```
      Task: Execute wave task T{task.id}: {task.name}
      Agent: mysd-executor
      Context: {
        ...(same as single mode plus):
        "worktree_path": "{worktree_dir}/T{task.id}",
        "branch": "mysd/{change_name}/T{task.id}-{task.slug}",
        "isolation": "worktree"
      }
      ```
    After all tasks in wave complete, proceed to merge step (same as Phase 6 SKILL.md)
    Then proceed to next wave.

Step 4: Post-Execution
- Run `mysd execute` (state transition — binary subcommand unchanged)
- Summary: tasks completed, tasks failed/blocked, `Next: /mysd:verify`

Copy to `plugin/commands/mysd-apply.md`.
