    ```go
    var langCmd = &cobra.Command{

        Args:  cobra.ExactArgs(1),
        RunE:  runLangSet,
    }

        langCmd.AddCommand(langSetCmd)
        rootCmd.AddCommand(langCmd)
    }
    ```

    **runLangRead:**
    1. Load mysd config via `config.Load(".")`
    2. Read openspec config via `spec.ReadOpenSpecConfig(".")`
    3. Output current settings:
       ```
       Language settings:
         mysd.yaml response_language: {cfg.ResponseLanguage || "(not set)"}
         openspec/config.yaml locale: {osCfg.Locale || "(not set)"}
       ```

    **runLangSet (ATOMIC dual-config write per Phase 5 D-09):**

    Uses **defer rollback pattern** (recommended by RESEARCH.md Pitfall 3 — safer on Windows than write-then-rename):

    1. Read current values:
       ```go
       cfg, err := config.Load(".")
       oldResponseLang := cfg.ResponseLanguage

       osCfg, err := spec.ReadOpenSpecConfig(".")
       oldLocale := osCfg.Locale
       ```

    2. Write mysd.yaml first (via Viper):
       ```go
       v := viper.New()
       configPath := filepath.Join(".", ".claude", "mysd.yaml")
       v.SetConfigFile(configPath)
       _ = v.ReadInConfig() // preserve other fields (Pitfall 1)
       v.Set("response_language", locale)
       if err := v.WriteConfig(); err != nil {
           if err2 := v.SafeWriteConfig(); err2 != nil {
               return fmt.Errorf("write mysd.yaml: %w", err2)
           }
       }
       ```

    3. Write openspec/config.yaml:
       ```go
       osCfg.Locale = locale
       if err := spec.WriteOpenSpecConfig(".", osCfg); err != nil {
           // ROLLBACK: restore mysd.yaml to old value
           v.Set("response_language", oldResponseLang)
           _ = v.WriteConfig()
           return fmt.Errorf("write openspec config (rolled back mysd.yaml): %w", err)
       }
       ```

    4. Print success:
       ```
       Language updated:
         response_language: {locale}
         locale: {locale}
       ```

    CRITICAL: The rollback defer pattern means if openspec/config.yaml write fails, mysd.yaml is restored. If mysd.yaml write fails first, we never attempt openspec write. This ensures both configs are always consistent (Phase 5 D-09).
