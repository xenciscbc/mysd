# Phase 03 Plan 02: UAT Checklist Package Summary

**One-liner:** UAT checklist data model with YAML frontmatter read/write and run_history append-not-overwrite semantics.

## What Was Built

The `internal/uat` package provides the data model and file I/O for UAT (User Acceptance Testing) checklists stored in `.mysd/uat/{change}-uat.md` files.

**Exported API:**
- `UATChecklist` — top-level struct with frontmatter fields
- `UATItem` — single UAT test item (id, description, status, notes, run_at)
- `UATSummary` — aggregate pass/fail/skip counts
- `UATRunResult` — historical run snapshot (run_at + summary)
- `NewUATChecklist(changeName, items)` — creates checklist with auto-assigned uat-N IDs and pending status
- `UATFilePath(baseDir, changeName)` — returns canonical `.mysd/uat/{change}-uat.md` path
- `WriteUAT(filePath, checklist)` — writes YAML frontmatter + markdown checkboxes, auto-creates directory, preserves history
- `ReadUAT(filePath)` — parses YAML frontmatter, returns zero-value on missing file

**File format produced by WriteUAT:**
```
---
spec-version: "1"
change: my-feature
generated: 2026-03-24T01:45:00Z
last_run: 2026-03-24T01:47:00Z
summary:
  total: 2
  pass: 0
  fail: 0
  skip: 0
results:
  - id: "uat-1"
    description: "User can see the login button"
    status: pending
run_history: []
---

## UAT Checklist: my-feature

_Generated by mysd verify. Run `/mysd:uat` to conduct acceptance testing._

### UI Acceptance Tests

- [ ] User can see the login button
```

## Decisions Made

| Decision | Rationale |
|----------|-----------|
| UATItem.Status is plain string, not spec.ItemStatus | UAT statuses ('pass'/'fail'/'skip') diverge from spec lifecycle ('pending'/'in_progress'/'done'/'blocked') — separate type avoids confusion |
| run_history captures the *existing* results on write | Preserves what was recorded in the file before the new run overwrites Results — enables before/after comparison |
| ReadUAT returns zero-value on missing file | Matches UAT-04 requirement: repeated runs should not fail on first use when no UAT file exists yet |
| WriteUAT always updates last_run to now | Tracks when the checklist was last regenerated, independent of when items were tested |
