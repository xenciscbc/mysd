Binary commands available (from Plan 03/04):
- `mysd execute --context-only` outputs JSON with change_name, must_items, tasks, pending_tasks, tdd_mode, execution_mode, agent_count

- `mysd plan --context-only` outputs JSON with change_name, phase, specs, design, model
- `mysd task-update <id> <status>` updates task status
- `mysd status` renders dashboard
- `mysd ff <name>` fast-forward state transitions
- `mysd ffe <name>` fast-forward through execute
- `mysd propose <name>` scaffold change
- `mysd capture --name <name>` guidance only, AI-side logic in SKILL.md
- `mysd init` create/display config
</interfaces>
